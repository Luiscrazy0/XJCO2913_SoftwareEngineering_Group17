## 🎨 电动滑板车租借平台：迈凯伦性能级视觉设计方案

### 1. 核心视觉理念：Performance-Driven (性能驱动)
* **品牌基因**：迈凯伦的“流线型（Flow）”与“碳纤维（Lightweight）”。
* **设计隐喻**：将每一个租借组件视作赛车零件。橙色不仅是色彩，更是“激发动力”的信号。
* **技术美学**：利用 3D 深度感模拟物理按键，打破扁平化的沉闷，增加操作的“仪式感”。

---

### 2. 视觉规范表 (Design Tokens)

| 类别 | 属性值 | 应用建议 |
| :--- | :--- | :--- |
| **主色 (Brand)** | `#FF6A00` | 核心 3D 按钮、高亮状态、电量进度条 |
| **背景 (Surface)** | `#0E0E0E` (主) / `#111827` (卡片) | 全局背景，使用微量渐变模拟金属质感 |
| **分割 (Divider)** | `#2D3748` (0.5px - 1px) | 模块间、列表项间的物理区分 |
| **主字体 (UI)** | `Inter`, `SF Pro Display` | 保持界面现代、清晰 |
| **数值字体 (Data)** | `Maple Mono NF CN` 或 `Roboto Mono` | **关键点**：用于时速、电量、坐标等数据，模拟仪表盘 |

---

### 3. 深度组件设计标准

#### 3.1 3D 机械感按钮 (The "Ignition" Button)
* **构造**：按钮不再是色块，而是一个有厚度的实体。
* **视觉**：利用 `box-shadow` 的垂直偏移（6px - 8px）营造高度。
* **反馈**：按下时 `translateY(4px)`，厚度缩减，模拟真实的开关手感。

#### 3.2 碳纤维卡片 (The Chassis Card)
* **边框**：`1px solid #374151`，边缘做极小的 `border-radius: 12px`。
* **背景**：在 `#111827` 基础上，可添加 2% 透明度的斜纹 SVG 图案，模拟碳纤维纹理。
* **间距**：严格执行 `gap: 24px`，确保每个模块之间有足够的“空气感”，避免视觉拥挤。

#### 3.3 仪表盘式筛选器 (Filtering System)
* **交互**：所有 Tab 切换增加 1px 的底部滑动橙色亮条。
* **区分**：不同车辆状态（Available / In Use）使用带发光效果（Blur）的小型圆点，模拟赛车仪表灯。

---

### 4. 关键页面布局方案

#### A. 车辆搜索页 (Scooter Discovery)
* **头部**：采用沉浸式设计，背景半透明磨砂（Glassmorphism），显示当前城市定位。
* **列表**：采用 Grid 布局，每辆车是一个“性能方框”。
    * **亮点**：车辆图片下方紧贴一层细灰线，线下展示“续航/电量/单价”三个数据列，使用等宽字体。

#### B. 预约模态框 (Booking Engine)
* **设计**：弹窗不再是纯白或纯黑，而是带有深蓝色调的 `#0E1322`。
* **细节**：确认按钮放置在底部中央，赋予最强的 3D 投影，作为操作的最终触点。
* **灰线应用**：取车点与还车点信息之间，用 `border-left: 1px solid #2D3748` 进行垂直切割。

---

### 5. 交互与动效 (Performance Animation)

1.  **启动加载 (Engine Start)**：进入页面时，橙色元素（如 Logo、按钮高光）先于内容 200ms 亮起。
2.  **悬停响应 (Hover)**：当鼠标滑过车辆卡片，卡片整体轻微上浮，橙色边框亮度提升 10%。
3.  **成功提示 (Victory Lap)**：预约成功后，屏幕顶部滑过一道橙色渐变光束，模拟赛车撞线。

---

### 6. 前端代码实现建议 (SCSS/CSS Variables)

```scss
:root {
  --mclaren-orange: #FF6A00;
  --track-black: #0E0E0E;
  --carbon-gray: #1F2937;
  --divider-line: #2D3748;
  
  /* 3D 阴影预设 */
  --shadow-3d: 0 6px 0 #C95000, 0 8px 24px rgba(0,0,0,0.55);
}

.scooter-card {
  border: 1px solid var(--divider-line);
  background: var(--carbon-gray);
  transition: transform 0.3s cubic-bezier(0.34, 1.56, 0.64, 1); /* 模拟弹簧阻尼感 */
}
```

### 💡 极客建议 (Plus Item)
考虑到你目前在进行的 **MVP 开发**，建议先封装一个名为 `MclarenProvider` 的全局主题，将 **灰线 (Gray Line)** 和 **3D 投影 (Shadows)** 作为全局 CSS 变量。这样在 Sprint 1 交互演示时，只需调整一个变量，就能让整个应用的“机甲感”瞬间拉满。
