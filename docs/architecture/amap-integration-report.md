## 使用说明

### 1. 获取API Key
1. 访问 [高德开放平台](https://lbs.amap.com/)
2. 注册账号并完成实名认证
3. 创建应用，获取JS API Key和Web API Key

### 2. 配置环境变量
在项目根目录的 `.env` 文件中配置：

```bash
# 高德地图API
VITE_AMAP_JS_KEY=your_js_api_key_here
VITE_AMAP_WEB_KEY=your_web_api_key_here
```

### 3. 启动项目
```bash
# 启动后端
cd backend
npm run start:dev

# 启动前端
cd frontend
npm run dev
```

### 4. 访问地图页面
- 开发环境: http://localhost:5176/map
- 测试页面: http://localhost:5176/src/test-amap.html

### 5. API端点
后端提供了以下API端点：
- `GET /amap/status` - 服务状态
- `GET /amap/geocode` - 地址转坐标
- `GET /amap/regeocode` - 坐标转地址
- `GET /amap/distance` - 距离计算
- `POST /amap/distances` - 批量距离计算
- `GET /amap/user-to-stations` - 用户到站点距离

## 技术债务和注意事项

### 1. API Key安全
- 当前使用环境变量管理，适合中小型项目
- 大型项目建议使用密钥管理服务（如AWS KMS、Azure Key Vault）

### 2. 使用限制
- 高德地图API有免费额度限制
- 需要监控使用量，避免超额费用
- 生产环境需要配置域名白名单

### 3. 浏览器兼容性
- 高德地图JS API支持现代浏览器
- 需要测试在目标用户浏览器中的兼容性
- 考虑提供降级方案（如静态地图）

### 4. 移动端优化
- 当前设计已考虑响应式
- 需要在实际移动设备上测试
- 考虑添加移动端特定功能（如摇一摇定位）
