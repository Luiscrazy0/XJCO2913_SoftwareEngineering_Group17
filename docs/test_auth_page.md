## 与后端API一致性验证

### 请求格式 ✅
- 登录请求：`{ email: string, password: string }`
- 注册请求：`{ email: string, password: string }`
- ✅ 与后端DTO完全一致

### 响应处理 ✅
- 登录响应：`{ access_token: string }`
- 注册响应：`{ id: string, email: string }`
- ✅ 直接处理后端原始数据，无包装层

### 错误处理 ✅
- 后端错误格式：NestJS BadRequestException
- 前端错误提取：`error.response?.data?.message`
- ✅ 错误消息正确显示
