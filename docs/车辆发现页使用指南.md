# 车辆发现页使用指南

## 概述
车辆发现页是电动滑板车租赁系统的核心功能页面，允许用户查看当前可用的车辆并进行预约。

## 访问方式

### 开发环境
1. 启动后端服务：
   ```bash
   cd backend
   npm run start:dev
   ```

2. 启动前端服务：
   ```bash
   cd frontend
   npm run dev
   ```

3. 访问页面：
   - 主页面（需要登录）：http://localhost:5173/scooters
   - 测试页面（无需登录）：http://localhost:5173/test-scooters

## 功能说明

### 1. 车辆列表显示
- 显示所有可用的电动滑板车
- 每辆车显示以下信息：
  - 车辆ID（缩短显示）
  - 位置信息
  - 状态标签（可用/不可用）
  - 预约按钮

### 2. 状态管理
页面有4种状态：
1. **加载中**：显示加载动画
2. **错误**：显示错误信息和重试按钮
3. **空状态**：没有可用车辆时的提示
4. **成功**：正常显示车辆列表

### 3. 过滤功能
- 自动过滤出状态为"AVAILABLE"的车辆
- 显示可用车辆数量统计
- 显示车辆总数和不可用数量

### 4. 预约功能
- 点击"立即预约"按钮触发预约事件
- 当前阶段显示提示信息
- 下一阶段将实现完整的预约流程

## 技术架构

### 前端技术栈
- **React 19**：UI框架
- **TypeScript**：类型安全
- **TanStack Query**：数据获取和状态管理
- **Tailwind CSS v4**：样式框架
- **Axios**：HTTP客户端

### 组件结构
```
src/
├── pages/
│   └── ScooterListPage.tsx      # 主页面
├── components/
│   └── ScooterCard.tsx          # 车辆卡片组件
├── api/
│   └── scooters.ts              # API封装
└── types/
    └── index.ts                 # 类型定义
```

### 数据流
1. 页面加载时调用 `scootersApi.getAll()`
2. TanStack Query管理数据状态
3. 过滤出可用车辆
4. 渲染ScooterCard组件列表
5. 用户交互触发相应事件

## 常见问题

### Q1: 页面显示"加载失败"怎么办？
**检查步骤：**
1. 确认后端服务是否运行（http://localhost:3000）
2. 检查网络连接
3. 查看浏览器控制台错误信息
4. 点击"重试"按钮

### Q2: 为什么看不到车辆？
**可能原因：**
1. 后端没有车辆数据
2. 所有车辆都处于"UNAVAILABLE"状态
3. API调用失败

### Q3: 如何添加测试车辆？
**方法：**
1. 通过后端API添加：
   ```bash
   curl -X POST http://localhost:3000/scooters \
     -H "Content-Type: application/json" \
     -d '{"location": "测试位置"}'
   ```

### Q4: 预约按钮点击没反应？
**说明：**
- 当前阶段预约功能只显示提示信息
- 完整的预约流程将在下一阶段实现
- 这是预期的行为
