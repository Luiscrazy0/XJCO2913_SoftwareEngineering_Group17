## 提交规范说明

### 提交类型
- `feat`: 新功能
- `fix`: 错误修复
- `docs`: 文档更新
- `style`: 代码样式/格式
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建/工具相关

### 提交格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

### 示例
```
feat(auth): 实现登录表单验证

- 添加邮箱格式验证
- 实现密码长度检查
- 添加实时错误提示
- 集成后端API调用

Closes #123
```

## 推荐提交顺序

1. 先提交基础框架和功能
2. 然后提交样式和交互优化
3. 接着提交错误处理和加载状态
4. 最后提交文档更新和测试验证
5. 根据需要提交修复和优化

这样的提交顺序可以：
- 保持每个提交的专注性
- 便于代码审查和回滚
- 清晰记录开发过程
- 方便问题追踪和定位
